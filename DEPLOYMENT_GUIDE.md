# 图书管理系统 - 网站部署指南

## 项目说明
这是一个纯前端的图书管理系统，包含完整的登录注册、图书管理、在线阅读等功能。由于是纯前端项目（无后端、无数据库），部署非常简单，有多种免费或低成本的部署方案。

## 快速部署方案（推荐）

### 方案1：GitHub Pages（完全免费）
**推荐指数：★★★★★**

**优点：** 完全免费、自动部署、无需维护、支持HTTPS、访问速度快
**缺点：** 无后端支持（但本项目是纯前端，不需要后端）

**部署步骤：**

1. **注册GitHub账号**
   - 访问 https://github.com 注册账号（如果已有账号请跳过）

2. **创建新仓库**
   - 登录GitHub后，点击右上角的"+"号，选择"New repository"
   - 仓库名称：`book-management-system`
   - 描述：`巴金三部曲图书管理系统`
   - 选择"Public"（公开仓库）
   - 勾选"Add a README file"
   - 点击"Create repository"

3. **上传项目文件**
   - 进入新创建的仓库
   - 点击"Add file" → "Upload files"
   - 上传项目中的 `index.html` 文件
   - 滚动到底部，点击"Commit changes"

4. **启用GitHub Pages**
   - 进入仓库的"Settings"页面
   - 点击左侧菜单中的"Pages"
   - 在"Source"选项中，选择"main"分支
   - 点击"Save"
   - 等待几分钟，页面会显示你的网站地址（通常是：`https://你的用户名.github.io/book-management-system/`）

5. **完成部署**
   - 复制生成的网址，分享给任何人即可访问

### 方案2：Netlify（完全免费）
**推荐指数：★★★★★**

**优点：** 完全免费、自动部署、全球CDN、支持HTTPS、部署速度快
**缺点：** 无后端支持（但本项目是纯前端，不需要后端）

**部署步骤：**

1. **注册Netlify账号**
   - 访问 https://app.netlify.com/signup 注册账号

2. **创建新站点**
   - 登录后，点击"Add new site" → "Deploy manually"

3. **上传项目文件**
   - 拖拽 `index.html` 文件到指定区域
   - 或点击"Browse"选择文件上传

4. **完成部署**
   - 上传完成后，Netlify会自动部署
   - 部署完成后会生成一个随机的域名（如：`https://xxxxxx.netlify.app`）
   - 可以点击"Site settings"自定义域名（可选）

### 方案3：Vercel（完全免费）
**推荐指数：★★★★★**

**优点：** 完全免费、自动部署、全球CDN、支持HTTPS、开发者友好
**缺点：** 无后端支持（但本项目是纯前端，不需要后端）

**部署步骤：**

1. **注册Vercel账号**
   - 访问 https://vercel.com/signup 注册账号

2. **导入项目**
   - 登录后，点击"Add New..." → "Project"
   - 选择"Import Git Repository"或"Upload"上传文件
   - 如果选择上传，直接拖拽 `index.html` 文件

3. **完成部署**
   - Vercel会自动部署项目
   - 部署完成后会生成一个域名（如：`https://xxxxxx.vercel.app`）

## 本地测试方法

在部署前，你可以先在本地测试网站功能：

1. **保存文件**
   - 将 `index.html` 文件保存到你的电脑上

2. **打开文件**
   - 双击 `index.html` 文件，它会在默认浏览器中打开
   - 或右键点击文件，选择"打开方式" → 选择浏览器

3. **测试功能**
   - 注册账号
   - 登录系统
   - 浏览图书
   - 在线阅读

## 高级部署方案

### 方案4：云服务器部署（付费）
**推荐指数：★★★☆☆**

**优点：** 完全可控、支持自定义域名、可扩展后端功能
**缺点：** 需要付费、需要技术维护

**部署步骤：**

1. **购买云服务器**
   - 推荐平台：阿里云、腾讯云、华为云等
   - 选择最低配置即可（1核2G内存，足够纯静态网站使用）
   - 价格：约50-100元/月

2. **安装Web服务器**
   - 连接服务器（使用SSH）
   - 安装Nginx：`sudo apt update && sudo apt install nginx`
   - 启动Nginx：`sudo systemctl start nginx`

3. **上传文件**
   - 将 `index.html` 上传到 `/var/www/html/` 目录
   - 或使用SFTP工具（如FileZilla）上传文件

4. **配置域名**（可选）
   - 购买域名（约50元/年）
   - 将域名解析到服务器IP
   - 配置SSL证书（Let's Encrypt免费证书）

## 注意事项

1. **关于数据存储**
   - 本系统使用浏览器本地存储（localStorage）保存用户数据
   - 用户数据只存储在用户自己的浏览器中，不会上传到服务器
   - 不同设备或浏览器之间数据不共享

2. **关于安全性**
   - 由于是纯前端系统，用户密码存储在本地，相对安全
   - 但不建议在生产环境中存储敏感信息

3. **自定义配置**
   - 可以修改 `index.html` 文件中的图书数据，添加更多书籍
   - 可以修改样式、添加新功能等

4. **移动端适配**
   - 系统已完全支持移动端访问
   - 在手机、平板等设备上都能正常使用

## 技术支持

如果在部署过程中遇到问题，可以：

1. 检查文件是否正确上传
2. 确认部署平台的设置是否正确
3. 清除浏览器缓存后重新访问
4. 尝试使用不同的浏览器访问
